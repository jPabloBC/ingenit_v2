Sistema de Tracking de Visitas - IngenIT

🎯 Objetivo

Registrar visitas a cada proyecto de IngenIT para mostrar estadísticas en el dashboard administrativo.

📊 Proyectos Configurados

HL - Health & Lifestyle

URL: hl.ingenit.cl

Código: hl

MT - Mantenimiento TI

URL: mt.ingenit.cl

Código: mt

WS - Web Services

URL: ws.ingenit.cl

Código: ws

Main - IngenIT Principal

URL: ingenit.cl

Código: main

🛠️ Implementación en Proyectos

1. JavaScript/HTML (para sitios estáticos)

<script> // Agregar al final del <body> de cada página async function trackPageVisit() { try { await fetch('https://ingenit.cl/api/track-visit', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ project_code: 'hl', // Cambiar según proyecto project_url: window.location.href }) }); } catch (e) { console.warn('Tracking failed:', e); } } // Ejecutar al cargar la página window.addEventListener('load', trackPageVisit); </script>

2. React/Next.js (para aplicaciones)

// En el componente principal o layout import { useEffect } from 'react'; export default function Layout({ children }) { useEffect(() => { // Track visit cuando se monta el componente fetch('https://ingenit.cl/api/track-visit', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ project_code: 'hl', // Cambiar según proyecto project_url: window.location.href }) }).catch(e => console.warn('Tracking failed:', e)); }, []); return <div>{children}</div>; }

3. PHP (para sitios dinámicos)

<?php // Agregar al inicio de cada página principal function trackVisit($projectCode) { $data = [ 'project_code' => $projectCode, 'project_url' => 'https://' . $_SERVER['HTTP_HOST'] . $_SERVER['REQUEST_URI'] ]; $context = stream_context_create([ 'http' => [ 'method' => 'POST', 'header' => 'Content-Type: application/json', 'content' => json_encode($data) ] ]); @file_get_contents('https://ingenit.cl/api/track-visit', false, $context); } // Ejecutar trackVisit('hl'); // Cambiar según proyecto ?>

📈 Ver Estadísticas

Las estadísticas se muestran en el dashboard administrativo:

URL: http://localhost:3000/admin/dashboard

🗄️ Configuración de Base de Datos

Para configurar la tabla en Supabase, ejecuta este SQL en el editor SQL:

-- 1. Crear tabla CREATE TABLE project_visits ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), project_code text NOT NULL, project_url text NOT NULL, visitor_ip text, user_agent text, referrer text, created_at timestamptz DEFAULT now(), date_only date GENERATED ALWAYS AS (created_at::date) STORED ); -- 2. Crear índices CREATE INDEX idx_project_visits_project_code ON project_visits(project_code); CREATE INDEX idx_project_visits_date ON project_visits(date_only); -- 3. Configurar RLS ALTER TABLE project_visits ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow anonymous insert" ON project_visits FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "Allow admin read" ON project_visits FOR SELECT TO authenticated USING (true); -- 4. Crear vista de estadísticas CREATE VIEW project_visit_stats AS SELECT project_code, project_url, COUNT(*) as total_visits, COUNT(DISTINCT visitor_ip) as unique_visitors, COUNT(CASE WHEN date_only = CURRENT_DATE THEN 1 END) as today_visits, COUNT(CASE WHEN date_only >= CURRENT_DATE - INTERVAL '7 days' THEN 1 END) as week_visits, COUNT(CASE WHEN date_only >= CURRENT_DATE - INTERVAL '30 days' THEN 1 END) as month_visits, MAX(created_at) as last_visit FROM project_visits GROUP BY project_code, project_url ORDER BY total_visits DESC;
